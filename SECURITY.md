# Security Policy

## Supported Versions

This project is maintained as a single continuously updated version. Security fixes are applied to the latest version available in the repository.

| Version | Supported          |
| ------- | ------------------ |
| Latest   | :white_check_mark: |
| Older versions   | :x:                |

## Reporting a Vulnerability

The PSHS Demonlist team takes security issues seriously and appreciates responsible disclosure.

If you discover a security vulnerability, please do not open a public GitHub issue, as doing so could expose the vulnerability before it can be fixed.

Instead, report the issue privately by contacting the project maintainers through one of the following methods:

* GitHub private security advisory
* Direct contact with a repository maintainer

When reporting a vulnerability, please include:

* A description of the issue
* Steps to reproduce the vulnerability
* The potential impact of the issue
* Any screenshots, proof-of-concept code, or other supporting information

### What to Expect

* An acknowledgment of your report within 7 days, whenever possible.
* An investigation of the reported issue.
* Updates regarding the status of the report if additional information is needed.
* Notification when the issue has been resolved.

### Accepted Vulnerabilities

If a vulnerability is confirmed, the maintainers will work to develop and release a fix as soon as reasonably possible. Credit may be given to the reporter unless they request otherwise.

### Declined Reports

Reports may be declined if they:

* Do not represent a genuine security vulnerability
* Cannot be reproduced
* Depend on unrealistic attack scenarios
* Are related to third-party services outside the scope of this project

We appreciate all good-faith reports and the efforts of community members who help keep the project secure.
